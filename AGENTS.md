# Agent Instructions for OpenCode Only

> **هذه التعليمات مخصصة لبرنامج OpenCode والنماذج (models) المربوطة به فقط.**
> **لا تنطبق على أي برامج أو نماذج أخرى.**
> لكل برنامج فروعه وقواعده المستقلة.

---

## الهدف الأساسي

يجب أن تظل كل تعديلات OpenCode ظاهرة للمستخدم داخل **GitHub Desktop** قبل عمل
`commit` و`push`. يعمل OpenCode على الفرع الأساسي `main` مباشرةً، دون إنشاء أي
فرع مخصّص أو `worktree` أو clone مؤقت.

## مكان العمل والفرع

- اعمل داخل النسخة الأساسية فقط:
  `/Users/mac/Documents/GitHub/FrameID`
- اعمل على الفرع `main` مباشرةً؛ ممنوع إنشاء فروع أو `worktree` أو clone مؤقت
  أو العمل من مسار آخر.
- إذا وجدت تعديلات غير مكوميتة، حافظ عليها ولا تعمل reset أو stash أو checkout
  لأي ملف إلا بطلب صريح من المستخدم.
- قبل أي تعديل، اعرض الحالة باستخدام `git status --short --branch`.

## سير العمل الإلزامي

1. قبل التعديل، اعرض الحالة باستخدام `git status --short --branch`.
2. عدّل الملفات المطلوبة واختبرها.
3. بعد انتهاء التعديل، اعرض ملخصًا ونتيجة الاختبارات ثم **توقف للمراجعة**.

## الرفع بعد إنهاء المهمة

- بعد إنهاء المهمة المطلوبة (تعديل + اختبار) ومراجعة المستخدم للملفات تحت
  تبويب **Changes** في GitHub Desktop، يقوم OpenCode تلقائيًا بـ:
  1. `git add` للملفات الخاصة بالطلب/المهمة الحالية فقط (لا يضمّن ملفات
     لا علاقة لها بالمهمة).
  2. `git commit -m "<عنوان المهمة>"`.
  3. `git push origin main` لرفع التعديلات إلى `origin/main`.
- هذا السلوك خاص بـOpenCode فقط؛ أي برنامج/نموذج آخر لا يتبعه.
- لا ترفع إلا بعد انتهاء المهمة فعلًا ووجود تعديلات حقيقية تخصّ الطلب.
- لا تنشئ commit تجريبيًا فقط لاختبار الرفع.

## GitHub Desktop

- قبل الـcommit: تظهر الملفات في تبويب **Changes** على فرع `main`.
- بعد الـcommit: تنتقل التعديلات إلى تبويب **History**؛ هذا سلوك طبيعي.
- بعد الـpush: يظهر الفرع والـcommits على GitHub.
- يجب أن يكون GitHub Desktop مفتوحًا على نفس المسار الأساسي المذكور أعلاه،
  والفرع الحالي `main`.

## الأمان

- ممنوع وضع GitHub token أو كلمة مرور داخل رابط `origin`.
- يجب أن يكون رابط `origin` بالشكل التالي فقط:
  `https://github.com/badrabdoph1/FrameID.git`
- استخدم GitHub Desktop أو مدير بيانات الاعتماد للمصادقة.

## عناوين الـcommits

- عنوان الـcommit يطابق عنوان الطلب أو يصف التغيير بوضوح.
- لا تنشئ commit تجريبيًا فقط لاختبار الرفع.
