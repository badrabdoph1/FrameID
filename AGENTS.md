# Agent Instructions for opencode Only

> **هذه التعليمات مخصصة لبرنامج opencode والنماذج المربوطة به فقط**  
> **لا تنطبق على أي برامج أو نماذج أخرى**

---

## Branch Workflow (القاعدة الافتراضية الإلزامية)

### 1. في بداية أي عمل / مهمة / محادثة / إصلاح / طلب:
- **اشتغل على فرع `اوبن-كود` فقط**
- لو الفرع مش موجود: `git checkout -b اوبن-كود main`
- لو موجود: `git checkout اوبن-كود && git pull origin اوبن-كود`

### 2. في نهاية كل مهمة / محادثة / إصلاح / طلب:
- **أولاً: ادمج مع الفرع الرئيسي**
- **ثانياً: ارفع للـ remote**

## Merge & Push Procedure (إلزامي بالتسلسل)

1. تأكد إن كل التعديلات مكوميتة على `اوبن-كود`
2. `git checkout main`
3. `git pull origin main`
4. `git merge اوبن-كود`
5. `git push origin main` — برسالة مطابقة لعنوان الجلسة/الطلب
6. `git checkout اوبن-كود`
7. `git push origin اوبن-كود` (اختياري: عشان الفرع يبقى محدث على GitHub)

---

## ملاحظات مهمة

- **ممنوع** العمل مباشرة على `main`
- **ممنوع** عمل push لـ `main` من غير دمج من `اوبن-كود`
- كل جلسة/مهمة = دورة كاملة: ابدأ على `اوبن-كود` → اختم بالدمج والرفع لـ `main`
- عنوان الكوميت/الـ PR = عنوان الجلسة أو الطلب
