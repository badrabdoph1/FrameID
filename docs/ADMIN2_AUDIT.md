# Admin2 — مراجعة وإعادة هيكلة لوحة الأدمن الرئيسية

## نطاق العمل

هذه المرحلة تخص لوحة الأدمن الرئيسية فقط، وليست لوحة العميل. الهدف هو تقليل التكرار، توحيد أنماط العرض والتعديل، وتجميع المهام داخل Workspaces واضحة بدون حذف أي وظيفة حالية.

## الملاحظات الأولية

- توجد مكونات متشابهة تُعرّف داخل الصفحات نفسها مثل بطاقات المؤشرات، حالات الفراغ، الشارات، الروابط السريعة وPanels.
- بعض الصفحات تعرض أكثر من نقطة دخول لنفس المهمة، مثل الإجراءات في رأس الصفحة ثم روابط سريعة داخل المحتوى.
- بعض الأقسام تعرض أكثر من إجراء مباشر لكل عنصر بدل قائمة إجراءات موحدة.
- نماذج التعديل تُعرض أحيانًا داخل نفس الصفحة بدل مسار View → Edit واضح.
- يوجد خلط بين الإجراءات اليومية والإعدادات المتقدمة داخل نفس الواجهة.
- توجد مصطلحات وحالات تقنية بالإنجليزية تحتاج ترجمة عرض عربية موحدة.
- توجد صفحات Workspace بدأت بالفعل لكنها لا تستخدم Design System واحدًا بعد.

## Workspaces المستهدفة

1. الرئيسية
2. العملاء
3. الاشتراكات والمدفوعات
4. المحتوى
5. النظام

## المرجع الأول

قسم الباقات هو Reference Implementation الأول. يجب أن يحقق:

- عرض كل باقة مرة واحدة فقط.
- زر إجراء رئيسي واحد لإضافة باقة.
- زر تعديل واحد لكل باقة.
- قائمة إجراءات موحدة للحالة والأرشفة.
- عرض السعر، الحالة، الترتيب وعدد المشتركين.
- عدم عرض نموذج التعديل إلا عند الطلب.
- دعم ممتاز للهاتف وسطح المكتب.
- استخدام مكونات قابلة لإعادة الاستخدام لباقي الأقسام.

## قواعد التنفيذ

- لا حذف لأي وظيفة قبل مراجعة استخدامها وتأثيرها.
- الحفاظ على الصلاحيات والروابط والبيانات وAPIs.
- لا إضافة Features جديدة خلال Admin2.
- أي Technical Debt آمن يُصلح أثناء المرحلة، وغير الآمن يوثق لمرحلة مستقلة.
- لا يبدأ القسم التالي قبل اعتماد القسم الحالي كمرجع قياسي.
