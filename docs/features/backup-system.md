# نظام النسخ الاحتياطي

الواجهة تعرض DATABASE وFULL اليدوية، النسخ التلقائي، السجلات، والإحصائيات، ومسار «ذهاب/عودة». كل إجراء يستدعي Pipeline نفسها. «ذهاب» ينشئ FULL وينتظر Remote Verify، و«عودة» تستعيد آخر FULL من GitHub حتى بعد فقد قرص Railway.

السياسة ثابتة: DATABASE كل 12 ساعة/20 نسخة، وFULL كل 48 ساعة/10 نسخ.

يستخدم Scheduler الداخلي وGitHub Actions حجزًا ذريًا واحدًا لموعد التشغيل. الفترة تُحسب بالساعات من موعد التشغيل السابق، لذلك تظل 48 ساعة صحيحة عند الانتقال بين الشهور ولا تنشأ نسخة مكررة إذا وصل المشغّلان معًا.

عند حذف قاعدة Railway وإنشاء قاعدة جديدة، لا يبدأ النسخ التلقائي فورًا فوق النسخ السليمة. زر «عودة» يتجاوز أي FULL فارغة نشأت بعد الحذف، ثم لا يعلن النجاح إلا إذا طابقت أعداد العملاء والمواقع والوسائط Manifest النسخة المختارة.
