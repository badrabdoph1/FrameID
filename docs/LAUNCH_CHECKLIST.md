# FrameID Launch Checklist

هذه القائمة يجب مراجعتها قبل اعتبار FrameID جاهزًا للإطلاق العام والأرشفة في Google. لا تعتبر مرحلة الإطلاق مكتملة إلا بعد نجاح كل اختبارات الجودة الأساسية بنسبة 100%.

## 1. الدومين والاستضافة

- [ ] تأكيد الدومين النهائي المستخدم في الإنتاج.
- [ ] ربط الدومين بالاستضافة النهائية.
- [ ] تفعيل HTTPS على الدومين الأساسي.
- [ ] التأكد من تحويل `www` إلى الدومين الأساسي أو العكس حسب القرار النهائي.
- [ ] التأكد أن `NEXT_PUBLIC_APP_URL` أو إعدادات URL في البيئة تشير للدومين الصحيح.
- [ ] فتح الصفحة الرئيسية من الموبايل والكمبيوتر على الدومين النهائي.

## 2. Google Search Console

- [ ] إضافة الدومين إلى Google Search Console.
- [ ] تنفيذ طريقة التحقق المناسبة.
- [ ] إرسال `https://frameid.app/sitemap.xml` بعد استبدال الدومين إذا تغيّر.
- [ ] مراجعة Coverage / Indexing بعد أول crawl.
- [ ] التأكد أن صفحات `/admin`, `/dashboard`, `/login`, `/forgot-password`, `/reset-password`, `/api` غير مؤرشفة.

## 3. Google Analytics

- [ ] تحديد هل سيتم استخدام Google Analytics أم لا.
- [ ] إذا سيتم استخدامه: إضافة `NEXT_PUBLIC_GA_ID` في بيئة الإنتاج.
- [ ] اختبار ظهور الزيارات في Realtime.
- [ ] عدم تفعيل Analytics بدون تحديث سياسة ملفات تعريف الارتباط إذا تغيّر نوع التتبع.

## 4. SEO وRich Results

- [ ] فتح الصفحة الرئيسية واختبار title وmeta description.
- [ ] اختبار Open Graph Preview عند مشاركة الرابط في واتساب وفيسبوك.
- [ ] اختبار Twitter/X Card preview.
- [ ] اختبار Structured Data باستخدام Google Rich Results Test.
- [ ] التأكد من وجود Organization Schema.
- [ ] التأكد من وجود WebSite Schema.
- [ ] التأكد من وجود SoftwareApplication Schema عند إطلاق المنصة كـ SaaS.
- [ ] التأكد من وجود FAQ Schema فقط إذا كان قسم الأسئلة ظاهرًا في الصفحة.
- [ ] التأكد من عدم وجود محتوى مكرر أو صفحات ضعيفة داخل sitemap.

## 5. Robots وSitemap

- [ ] فتح `/robots.txt` والتأكد من منع الصفحات الداخلية.
- [ ] فتح `/sitemap.xml` والتأكد من وجود الصفحات العامة فقط.
- [ ] التأكد أن sitemap لا يحتوي صفحات noindex.
- [ ] تحديث تاريخ `lastModified` عند تغييرات الإطلاق المهمة.

## 6. البريد الإلكتروني

- [ ] إعداد متغيرات SMTP أو مزود البريد المستخدم في الإنتاج.
- [ ] اختبار إرسال بريد استعادة كلمة المرور.
- [ ] اختبار وصول البريد إلى Gmail.
- [ ] اختبار عدم دخول البريد إلى Spam قدر الإمكان.
- [ ] إضافة SPF/DKIM/DMARC للدومين إذا كان البريد من دومين FrameID.
- [ ] إضافة البريد الرسمي للدعم داخل إعدادات المنصة والصفحات القانونية عند اعتماده.

## 7. التسجيل وتسجيل الدخول

- [ ] اختبار إنشاء حساب جديد من `/signup`.
- [ ] اختبار إنشاء الموقع والرابط تلقائيًا بعد التسجيل.
- [ ] اختبار تسجيل الدخول من `/login`.
- [ ] اختبار التحويل الصحيح بعد تسجيل الدخول.
- [ ] اختبار رسالة الخطأ عند بيانات غير صحيحة.
- [ ] اختبار تسجيل الخروج إن وجد.

## 8. استعادة كلمة المرور

- [ ] اختبار `/forgot-password` ببريد صحيح.
- [ ] اختبار `/forgot-password` ببريد غير موجود دون كشف وجود الحساب.
- [ ] اختبار رابط `/reset-password`.
- [ ] اختبار صلاحية token وانتهاء صلاحيته.
- [ ] اختبار تغيير كلمة السر ثم تسجيل الدخول بالكلمة الجديدة.

## 9. الدفع والاشتراكات

- [ ] تحديد آلية الدفع النهائية.
- [ ] اختبار إنشاء طلب دفع أو مراجعة الدفع يدويًا حسب النظام الحالي.
- [ ] التأكد أن الاشتراك لا يتفعل إلا بعد مراجعة الدفع واعتماده.
- [ ] اختبار حالة الحساب قبل التفعيل وبعد التفعيل.
- [ ] اختبار انتهاء الاشتراك أو تعطيله.
- [ ] التأكد من وضوح سياسة عدم الاسترداد بعد التفعيل إلا للحالات الموضحة في الشروط.

## 10. النسخة الاحتياطية والاستعادة

- [ ] تشغيل اختبار نسخة احتياطية لقاعدة البيانات.
- [ ] تشغيل اختبار نسخة احتياطية للملفات/uploads إن وجدت.
- [ ] اختبار أمر الاستعادة في بيئة غير إنتاجية.
- [ ] التأكد من مكان حفظ النسخ الاحتياطية وصلاحيات الوصول.
- [ ] توثيق خطة استعادة الطوارئ.

## 11. PWA

- [ ] فتح الموقع من Chrome على Android واختبار install prompt.
- [ ] فتح الموقع من Safari على iOS واختبار Add to Home Screen.
- [ ] التأكد من ظهور الأيقونة الصحيحة.
- [ ] اختبار manifest من DevTools.
- [ ] اختبار screenshots الخاصة بـ PWA.
- [ ] اختبار فتح لوحة العميل من shortcut.

## 12. الموبايل والتوافق

- [ ] اختبار الصفحة الرئيسية على شاشة صغيرة 360px.
- [ ] اختبار الصفحة الرئيسية على iPhone/Safari.
- [ ] اختبار الصفحة الرئيسية على Android/Chrome.
- [ ] اختبار الهيدر والـ CTA والـ sticky CTA.
- [ ] اختبار صفحة القوالب على الموبايل.
- [ ] اختبار صفحات الخصوصية والشروط والكوكيز على الموبايل.
- [ ] اختبار لوحة التحكم الأساسية على الموبايل.

## 13. الأداء وCore Web Vitals

- [ ] تشغيل Lighthouse على الصفحة الرئيسية.
- [ ] مراجعة LCP.
- [ ] مراجعة CLS.
- [ ] مراجعة INP.
- [ ] التأكد من lazy loading للصور غير الحرجة.
- [ ] التأكد من عدم تحميل سكربتات غير ضرورية قبل التفاعل.
- [ ] مراجعة حجم الصور الخارجية.
- [ ] مراجعة font loading وعدم حدوث layout shift.

## 14. Accessibility

- [ ] اختبار التنقل بالكيبورد.
- [ ] اختبار focus states للأزرار والروابط.
- [ ] مراجعة تباين الألوان.
- [ ] التأكد من وجود labels لكل input.
- [ ] التأكد من alt text للصور المهمة.
- [ ] عدم استخدام أيقونات فقط بدون نص مفهوم.
- [ ] اختبار الصفحة الرئيسية بامتداد Lighthouse Accessibility.

## 15. الصفحات القانونية

- [ ] مراجعة سياسة الخصوصية.
- [ ] مراجعة شروط الاستخدام.
- [ ] مراجعة سياسة ملفات تعريف الارتباط.
- [ ] إضافة البريد الرسمي للدعم عند اعتماده.
- [ ] إضافة بيانات الشركة القانونية عند توفرها.
- [ ] التأكد من ظهور الروابط القانونية في Footer.

## 16. اختبارات الجودة النهائية

يجب تشغيل الأوامر التالية قبل الإطلاق:

```bash
npm run typecheck
npm run lint
npm run test
npm run build
```

لا يتم اعتماد الإطلاق إلا إذا نجحت كل الأوامر بدون أخطاء.

## 17. قرار الإطلاق

- [ ] كل الاختبارات ناجحة.
- [ ] الدومين يعمل.
- [ ] التسجيل يعمل.
- [ ] استعادة كلمة المرور تعمل.
- [ ] الصفحات العامة قابلة للأرشفة.
- [ ] الصفحات الداخلية ممنوعة من الأرشفة.
- [ ] النسخة الاحتياطية مجربة.
- [ ] الموبايل مجرب.
- [ ] Preview المشاركة يعمل.
- [ ] تم إرسال sitemap إلى Google Search Console.

بعد إكمال هذه القائمة يمكن الانتقال إلى مرحلة تحسين رحلة العميل والتحويل Conversion.
