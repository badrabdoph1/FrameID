# اكتمال مركز التواصل الموحّد

**التاريخ:** 2026-07-18
**الحالة:** التنفيذ الوظيفي مكتمل؛ تطبيق النشر الفعلي يتطلب قاعدة الإنتاج وجدولة مسارات التشغيل.

## المراحل المنفذة

1. **Application/Query Layer:** عقود بحث وعزل Tenant، Inbox العميل والأدمن، Badge موحد، وRead Cursors حقيقية.
2. **تجربة العميل:** قائمة متجاوبة، إنشاء طلب، Timeline، ردود، مسودة محلية، معاينة صور وحذفها قبل الإرسال، قراءة الرسائل، وإغلاق/إعادة فتح الطلب بعد الحل.
3. **تجربة الأدمن:** البحث والفلاتر، صفحة المحادثة، الرد، الملاحظات الداخلية، الحالة، الأولوية، التعيين، والسياقات العامة.
4. **المرفقات:** تحقق decoding وmagic content، إعادة ترميز WebP، تخزين خاص، checksum، صلاحية تنزيل، تنظيف orphan files، ومنع وصول العميل للملاحظات الداخلية.
5. **الحملات:** جمهور الكل/التجريبي/المشترك/المنتهي/الصريح، جدولة، Audience snapshot، تسليم In-App، إحصاء التسليم، وسحب مدقق.
6. **Delivery:** Outbox worker بlease وretry/backoff وidempotent attempts وdead-letter، مع In-App Adapter وCron endpoint محمي.
7. **التكامل والترحيل:** Bridge عام للإشعارات وCustomerRequest وSupportCase والحملات؛ ربط الدفع وحذف الحساب وحل البلاغات والرسالة الفردية والجماعية؛ Backfill قابل للإعادة ومصالحة تشغيلية.
8. **التشغيل:** Runbook، endpoint للمصالحة، متغير تخزين خاص، واختبارات PostgreSQL معزولة للـschema النهائي والـbackfill.

## قرارات معمارية أثناء التنفيذ

- بقي Communication Core بلا imports من Billing أو Services أو Sites؛ التكامل عند Composition Roots وبـContext Reference opaque.
- الكتابة القديمة مؤقتة في نقاط الانتقال فقط، والنواة هي المصدر الجديد للـInbox. تمنع Context/idempotency إعادة الترحيل.
- مرفقات الإصدار الأول صور فقط. فك الترميز وإعادة التشفير هو فحص المحتوى الحالي؛ Antivirus خارجي يؤجل حتى دعم ملفات عامة.
- الرسالة الفردية `DIRECT` وقابلة للرد، أما الإرسال الجماعي فيستخدم Conversation واحدة + Audience rows.
- منشئ الرسالة يُعلّم قارئًا لها داخل المعاملة نفسها، لمنع احتساب رسالته كغير مقروءة.

## التحقق

- اختبارات نطاق التواصل: 23 ملفًا، 101 اختبار، كلها ناجحة.
- ESLint الكامل: صفر أخطاء؛ توجد warnings قديمة خارج النطاق.
- Prisma schema النهائي: نجح `db push` على PostgreSQL 16 مؤقت ومعزول.
- Backfill على القاعدة النظيفة: نجح وأعاد صفرًا لكل المصادر الأربعة.
- `migrate deploy` من قاعدة فارغة توقف قبل Communication عند migration تاريخية `20260708060000_add_payment_system` بسبب استخدام قيمة Enum جديدة داخل نفس transaction (`55P04`).
- TypeScript والبناء يصلان إلى فحص الأنواع ثم يتوقفان في ملفات Cleanup المتزامنة غير التابعة للمركز؛ لا توجد أخطاء TypeScript في ملفات التواصل.
- الاختبارات الكاملة للمشروع: 476 ناجحة و27 فاشلة؛ الإخفاقات خارج نطاق التواصل وتشمل قاعدة غير متاحة وعقودًا قديمة لتغييرات متزامنة أخرى.

## ما لم يُفعّل الآن

- Email/Push/SMS: لا توجد قنوات مطلوبة أو تفضيلات فعلية بعد؛ أُعدت نقاط Adapter وcategory/group keys فقط.
- Saved Views وQuick Replies وAutomation وAnalytics وAI/Search body: مؤجلة وفق التقرير المعتمد إلى Phase 2 بعد وجود استخدام وبيانات حقيقية.
- Object storage متعدد العقد: التخزين المحلي الخاص صالح مع persistent volume؛ يضاف S3-compatible adapter قبل التوسع الأفقي.
- قطع الجداول القديمة: لا يتم قبل backfill إنتاجي، reconciliation نظيف، وفترة مراقبة.

## بوابة النشر

1. معالجة migration التاريخية أو استخدام مسار النشر الآمن المعتمد حاليًا للمشروع.
2. ضبط `CRON_SECRET` و`COMMUNICATION_ATTACHMENT_ROOT` على volume دائم.
3. جدولة delivery كل دقيقة وreconciliation كل خمس دقائق.
4. تشغيل backfill الإنتاجي حتى الصفر، ثم مراقبة المصالحة قبل إيقاف legacy writers.
