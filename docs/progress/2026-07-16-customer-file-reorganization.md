آخر تحديث: 2026-07-16

## الهدف

تقليل ملف العميل إلى أربع مساحات رئيسية، مع الحفاظ على الوظائف والروابط القديمة وإصلاح الأعطال التي تظهر أثناء التنفيذ.

## خط الأساس

- الفرع: `main`.
- قبل التعديل البرمجي: `tests/admin-customer-workspace.test.tsx` و`tests/admin-customers-table.test.tsx` و`tests/admin-navigation-contract.test.tsx` = 9 اختبارات ناجحة.
- لا توجد تغييرات مستخدم سابقة في شجرة العمل عند بدء المهمة.

## ملاحظات مكتشفة

- ملف العميل الحالي يعرض 5 بطاقات كبيرة، شريط إجراءات مزدحم، بطاقة كلمة مرور، ثم 8 تبويبات.
- `sendNotificationAction` ينتظر `notificationType` بينما العميل يرسل `type`.
- زر «نسخ الرابط» الحالي عبارة عن رابط خارجي، فينسخ ويفتح الموقع في الوقت نفسه.
- نسخ كلمة المرور كان يستخدم رسالة نجاح خاصة برابط الموقع.
- نموذج الإشعار كان يمسح البيانات حتى عند فشل الإرسال.
- تسمية زر التأكيد كانت تعرض «تأكيد الحذف» لعمليات لا تحذف شيئًا.

## نقطة الاستكمال التالية

لا توجد خطوة تنفيذية متبقية. التغييرات محفوظة محليًا وغير مرفوعة أو مدمجة، وجاهزة لمراجعة المستخدم داخل Changes. خادم المعاينة وقاعدة PostgreSQL المحلية يعملان حاليًا لترك صفحة العميل مفتوحة للمراجعة.

## نتائج Task 3

- تجميع الموقع مع الملفات، والاشتراك مع المدفوعات، وكلمة المرور والجلسات مع التواصل والمتابعة.
- صغرت بطاقات المؤشرات إلى ارتفاع 60px على سطح المكتب، وصغرت بطاقة بيانات العميل ومركز الإجراءات دون تقليل أهداف اللمس الأساسية.
- اختبار تركيب المساحات الأربع ناجح، ولا تظهر كلمة المرور خارج «الدعم والحماية».

## نتائج Task 4

- يصل نوع الإشعار باسم `notificationType` الصحيح.
- تبقى بيانات نموذج الإشعار عند فشل الإرسال، ويظهر «جاري الإرسال…» أثناء التنفيذ.
- رسائل النسخ منفصلة لرابط الموقع وكلمة المرور، مع إرشاد واضح عند فشل الحافظة.
- حذف الملاحظة وتغيير كلمة المرور يمران عبر نافذة تأكيد، وتسمية زر التأكيد تطابق العملية.
- أضيفت تسميات حقول وأسماء أزرار تنزيل وحالات تركيز واضحة للعناصر المعدلة.

## نتائج Task 5 وTask 6

- الاختبارات المرتبطة: 6 ملفات، 24 اختبارًا ناجحًا.
- `npm run typecheck`: ناجح.
- `npm run lint`: ناجح بلا أخطاء؛ 50 تحذيرًا موجودًا خارج ملفات التغيير.
- `npm run build`: ناجح، ومسار `/admin/customers/[id]` بُني بنجاح.
- الاختبارات العامة مع وصول PostgreSQL المحلي: 339 ناجحًا و23 فشلًا في 16 ملفًا خارج قسم ملف العميل؛ لا يوجد فشل في اختبارات هذا التغيير.
- فحص فعلي بعميل محلي على 1440×900 و390×844: لا يوجد تمرير أفقي، شبكة التنقل 4 أعمدة على سطح المكتب و2×2 على الهاتف، وروابط التنزيل ظاهرة بحجم 44×44.
- اختُبر الانتقال إلى الاشتراك ثم زر الرجوع، فتح الرابط القديم `?tab=notes` داخل الدعم، وفتح وإلغاء تأكيد تغيير كلمة المرور دون تنفيذ تغيير فعلي.
- أغلقت المراجعة: تكرار التفعيل للحساب الموقوف، مزامنة الرابط مع المحتوى، دلالة روابط التنقل، أسماء المساحات، أهداف اللمس، وتمرير جدول المدفوعات.
