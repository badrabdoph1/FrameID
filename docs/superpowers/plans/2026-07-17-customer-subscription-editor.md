# خطة تنفيذ محرر اشتراك العميل

1. إضافة أنواع الباقات ومدخلات تعديل الاشتراك إلى نطاق إدارة العملاء.
2. كتابة اختبارات خدمة حساب المدة والتحقق من المدخلات أولًا.
3. إنشاء خدمة محرر الاشتراك ومستودع Prisma بمعاملة ذرية.
4. إضافة Server Action للتعديل مع سجل المشرف ومعالجة الأخطاء.
5. جلب الباقات النشطة في صفحة ملف العميل وتمريرها للواجهة.
6. إضافة نموذج «تعديل الاشتراك» المضغوط مع ملخص وخيارات مدة سهلة.
7. إضافة اختبار واجهة يغطي عميلًا بدون اشتراك وتسجيل دفعة اختياريًا.
8. تشغيل الاختبارات المستهدفة ثم typecheck وlint وbuild.
9. اختبار المسار داخل المتصفح بحساب عميل فعلي والتأكد من النتيجة بصريًا.

> تُترك الملفات غير مكوميتة للمراجعة وفق تعليمات المشروع.
