# تصميم توضيح الفرق بين لوحة الإدارة والموقع المنشور

## الهدف

يجب أن يفهم المصور خلال أول خمس ثوانٍ أنه داخل لوحة إدارة لا يراها العميل، وأن الموقع المنشور كيان مختلف يمكنه معاينته ونسخ رابطه وإرساله للعميل. التغيير تنظيمي ونصي وبصري فقط، ولا يغيّر منطق النشر أو المسارات أو قاعدة البيانات أو الخدمات.

## حدود التغيير

- الحفاظ على الشريط الجانبي ونسبة الاكتمال وخطوات التجهيز ونظام النشر الحالي.
- إعادة استخدام `DashboardViewModel` وحالة النشر ورابط الموقع ومنطق النسخ الحالي.
- عدم إضافة API أو جدول أو Route جديد.
- عدم تكرار أي Component أو Business Logic.
- الحفاظ على الاستجابة للموبايل وسطح المكتب، والوصول بلوحة المفاتيح، والأداء.

## ترتيب الصفحة الرئيسية

1. رسالة سياقية أعلى محتوى الصفحة توضّح: «أنت الآن داخل لوحة إدارة موقعك»، و«عملاؤك لن يروا هذه الصفحة».
2. كارت رئيسي بعنوان «موقعك المنشور» يكون أكبر عنصر بصري بعد الرسالة.
3. كارت الجاهزية الحالي دون تغيير في منطقه أو نسبته.
4. خطوات التجهيز الحالية والأزرار التالية دون حذف أي وظيفة.

## رسالة لوحة الإدارة

تظهر الرسالة بصورة بارزة للمستخدم الذي لم يؤكد فهمها من قبل. تحتوي على نص مباشر وزر إغلاق مسمى بوضوح. تُحفظ حالة الإقرار داخل نفس إعدادات `localStorage` المستخدمة لقائمة Onboarding الحالية. بعد الإقرار تظهر نسخة مدمجة غير مزعجة، ويمكن إبقاء السياق واضحًا دون إضافة شاشة ترحيب أو خطوة قبل Dashboard.

## كارت «موقعك المنشور»

يعرض الكارت:

- معاينة حقيقية للموقع المنشور داخل iframe غير تفاعلية ومحمّلة بتكاسل، وفق نمط المعاينات الموجود في المشروع.
- اسم الموقع.
- حالة النشر الحالية.
- عنوان «الرابط الذي سترسله لعملائك» والرابط العام النظيف.
- النص: «هذا هو الموقع الذي سيراه عملاؤك عند فتح الرابط. لوحة التحكم هذه لا يراها أي عميل.»
- زر «شاهد الموقع كما يراه العميل».
- زر «انسخ رابط الموقع لإرساله للعميل».

إذا كان الموقع غير منشور، لا تُحمّل صفحة عامة ستعيد 404. تظهر مساحة معاينة واضحة لحالة المسودة مع توجيه المستخدم إلى خطوة النشر الحالية، مع بقاء الحالة والرابط والمنطق كما هما.

## وضع معاينة المالك

زر المعاينة يفتح نفس المسار العام الحالي مع Query داخلي `ownerView=1`. لا يُضاف Route جديد.

تظهر رسالة المالك داخل الصفحة العامة فقط عندما تتحقق الشروط كلها:

1. يوجد `ownerView=1`.
2. توجد جلسة مستخدم صالحة.
3. الـslug المفتوح هو موقع المستخدم المسجل.

تعرض الرسالة: «أنت الآن تشاهد موقعك كما يراه العميل» ورابط «العودة إلى لوحة الإدارة».

الرابط المنسوخ والمشارك يظل بلا Query، لذلك لا تظهر الرسالة للزائر أو عند مشاركة الرابط. iframe داخل كارت Dashboard تستخدم الرابط النظيف، لذلك لا تعرض الرسالة. توليد Metadata يبقى مستقلًا ولا يتأثر بالعلامة، فلا يتغير SEO.

## إعادة استخدام المكونات والمنطق

- يُستخرج منطق النسخ وعرض أفعال الموقع إلى المكوّن الحالي المناسب بدل بقائه مكررًا بين الكارت والأزرار السفلية.
- تُستخدم قيم `siteTitle` و`siteUrl` و`statusLabel` و`isPublished` الموجودة في `DashboardViewModel`.
- يُعاد استخدام نمط iframe الموجود في `TemplateLivePreview` دون نسخ منطق Business أو إنشاء مصدر بيانات جديد.
- تُحدّث تسميات فتح الموقع في الأماكن المرتبطة برحلة Dashboard حتى تشير بوضوح إلى منظور العميل.

## حالات الخطأ والحواف

- فشل Clipboard لا يمنع باقي الصفحة، ويظل الزر قابلًا لإعادة المحاولة.
- الموقع المسودة لا يحمّل iframe عامة.
- مستخدم مسجل يفتح موقعًا لا يملكه لا يرى Banner المالك.
- زائر غير مسجل لا يرى Banner حتى لو أضاف Query يدويًا.
- الرابط المشارك لا يحتوي على `ownerView`.

## الاختبارات والتحقق

- اختبارات Component للنصوص الجديدة، الرابط النظيف، النسخ، ووضع المالك.
- اختبار قرار ظهور Banner للمالك وفشله للزائر أو لموقع مختلف.
- اختبار أن المسودة لا تحمّل iframe عامة.
- Typecheck وLint واختبارات المشروع، مع فصل أعطال Baseline القديمة غير المرتبطة في التقرير.
- مراجعة بصرية على سطح المكتب والموبايل والتأكد من إجابات أسئلة الخمس ثوانٍ:
  - أين أنا؟
  - ماذا سيرى العميل؟
  - ما الرابط الذي سأرسله؟
- التقاط صور قبل وبعد وإرفاقها بتقرير Pull Request.
