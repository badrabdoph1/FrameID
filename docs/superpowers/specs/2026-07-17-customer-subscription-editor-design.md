# تصميم محرر اشتراك العميل

## الهدف

إضافة أداة بسيطة داخل ملف العميل في مساحة «الاشتراك والدفع» تسمح للمشرف بتكوين اشتراك جديد أو تعديل الاشتراك الحالي، مع إبقاء حالة العميل والموقع وسجل الدفع متناسقة.

## تجربة الاستخدام

- يظهر زر «تعديل الاشتراك» حتى عندما لا يملك العميل اشتراكًا.
- يفتح الزر نموذجًا مدمجًا ومضغوطًا داخل نفس المساحة.
- يختار المشرف الباقة والحالة والمدة دون التعامل مع حقول قاعدة البيانات الخام.
- المدة تدعم: 30 يومًا، 90 يومًا، سنة، دائم، تاريخًا محددًا، أو زيادة/خصم عدد أيام من النهاية الحالية.
- يعرض النموذج ملخصًا واضحًا قبل الحفظ.
- خيار «تسجيل دفعة معتمدة» اختياري، ولا تظهر حقوله إلا عند تفعيله.
- تظل أزرار التفعيل والإلغاء وتمديد التجربة الحالية موجودة.

## قواعد المجال

- الباقة المختارة يجب أن تكون باقة نشطة.
- تسجيل دفعة يتطلب اشتراكًا بحالة «نشط»، ومبلغًا موجبًا، وطريقة دفع صحيحة.
- الاشتراك النشط أو التجريبي يجب أن ينتهي مستقبلًا، باستثناء الاشتراك الدائم.
- خصم أيام يؤدي إلى تاريخ منتهٍ يحول الحالة إلى «منتهي» بدل حفظ حالة نشطة غير صحيحة.
- حالة الاشتراك تتحكم في حالة العميل والموقع في المعاملة نفسها:
  - نشط: العميل نشط والموقع منشور.
  - تجريبي: العميل تجريبي والموقع منشور.
  - موقوف: العميل والموقع موقوفان.
  - منتهي/متأخر/ملغي: العميل منتهي والموقع غير منشور.
- كل تعديل ينشئ `SubscriptionChange` و`AuditLog`.
- عند اختيار تسجيل الدفع ينشأ `PaymentRequest` معتمد وسجل له باسم المشرف.

## المعمارية

- خدمة مستقلة تحسب المدة وتتحقق من المدخلات، لتكون قابلة للاختبار دون قاعدة بيانات.
- مستودع Prisma ينفذ تحديث الاشتراك والعميل والموقع والسجلات والدفع داخل معاملة واحدة.
- Server Action وحيد يستقبل النموذج، يتحقق من جلسة المشرف، ثم يستدعي الخدمة.
- الصفحة تجلب قائمة الباقات النشطة وتمُررها إلى مكوّن الاشتراك.

## التوافق

- لا تغيير على مخطط قاعدة البيانات.
- لا حذف لأي إجراء حالي.
- سجل المدفوعات الحالي سيعرض الدفعة اليدوية تلقائيًا بعد تحديث الصفحة.
- الاشتراك الدائم يمثل بنهاية بعيدة ثابتة لحماية عمليات دورة الحياة الحالية.

## الاختبارات

- حساب تواريخ المدد والزيادة والخصم والدائم.
- إنشاء اشتراك جديد وتعديل اشتراك قائم.
- منع دفع مع حالة غير نشطة أو مبلغ غير صالح.
- التحقق من ظهور المحرر وحقول الدفع الشرطية في الواجهة.
- فحص TypeScript وLint وBuild، ثم تجربة فعلية داخل المتصفح.
