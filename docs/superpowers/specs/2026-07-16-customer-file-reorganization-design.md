# تصميم إعادة تنظيم ملف العميل في الأدمن

## الهدف

تبسيط صفحة إدارة العميل التي تفتح من زر «فتح الملف» بحيث يفهمها عضو فريق غير تقني بسرعة، مع إبقاء صفحة قائمة العملاء الخارجية على تصميمها الحالي، والحفاظ على كل الوظائف والصلاحيات وإجراءات الخادم والروابط العميقة الموجودة.

## حدود النطاق

- تبقى `/admin/customers` بنفس الهيكل الحالي. يسمح فقط بتحسينات صغيرة مرتبطة بالوضوح أو عطل يظهر أثناء التنفيذ.
- تتركز إعادة التنظيم في `/admin/customers/[id]` ومكوناته.
- لا تتغير قاعدة البيانات أو مستودعات Prisma أو صلاحيات الأدمن.
- لا تحذف وظيفة: البحث، التفعيل، الإيقاف، التمديد، النشر، كلمة المرور، الجلسات، التواصل، الأرشفة والحذف كلها تبقى.

## هيكل المعلومات الجديد

تتحول الوجهات الثماني الحالية إلى أربع مساحات رئيسية فقط:

1. **الملخص والإجراءات**: بيانات العميل، المؤشرات المختصرة، الإجراءات اليومية، حالة الحساب، آخر النشاطات والانتقالات السريعة.
2. **الموقع والملفات**: إدارة الموقع والنشر والفتح والنطاقات وSEO والوسائط والبحث والتنزيل.
3. **الاشتراك والمدفوعات**: الاشتراك الحالي والتجربة والتمديد والتفعيل والإلغاء وسجل الاشتراكات والمدفوعات والإثباتات.
4. **الدعم والحماية**: كلمة المرور والجلسات والإشعارات والملاحظات ووسائل التواصل.

لا تنشأ تبويبات داخلية جديدة داخل هذه المساحات. تستخدم العناوين والفواصل فقط لتوضيح الأجزاء الطويلة.

## التوافق مع الروابط القديمة

تستخدم الواجهة القيم الأساسية `overview` و`site` و`billing` و`support`. تبقى القيم القديمة صالحة وتتحول داخليًا:

- `overview` ← `overview`
- `website` و`media` ← `site`
- `subscription` و`payments` ← `billing`
- `sessions` و`notifications` و`notes` ← `support`

بهذا تستمر الروابط الموجودة في صفحات الاشتراكات والتواصل ومشاكل العملاء في فتح السياق الصحيح، حتى قبل تحديثها إلى القيم الأساسية الجديدة.

## تركيب الصفحة

### الرأس والمؤشرات

- رأس مختصر باسم العميل وحالته والبريد ورابط الرجوع.
- خمسة مؤشرات مدمجة بارتفاع صغير: الحساب، التجربة، المواقع، الإيرادات والدعم.
- لا تستخدم بطاقات كبيرة أو مساحات رأسية زائدة.

### التنقل

- أربعة أزرار بعرض متساوٍ على سطح المكتب.
- شبكة 2×2 على الهاتف بدل شريط أفقي طويل.
- كل زر له اسم مباشر ووصف قصير، ويحفظ المساحة الحالية في عنوان URL.

### مركز الإجراءات

تظهر كل الإجراءات في مركز واحد، وتجمع داخل صفوف متقاربة:

- **الحساب والاشتراك**: تمديد التجربة، التفعيل أو التشغيل، الإيقاف، والانتقال لبيانات الدخول.
- **الموقع**: فتح الموقع، نسخ الرابط، ونشره أو إيقافه.
- **التواصل**: إشعار، بريد وواتساب.
- **إجراءات حساسة**: الأرشفة والحذف، مع تأكيد يشرح الأثر.

يبقى زر إعادة كلمة المرور موجودًا، لكنه ينقل المستخدم إلى مساحة «الدعم والحماية» حيث تظهر كلمة المرور والجلسات معًا، بدل توليد كلمة مرور غير معروضة داخل شريط سريع.

## حالات الواجهة والأعطال المشمولة

- تستخدم التأكيدات الحالية `AdminConfirmDialog` بدل إدخال تأكيدات متصفح جديدة.
- رسالة النجاح أو الفشل تذكر العملية التي تمت أو تعذرت.
- فشل النسخ إلى الحافظة يظهر رسالة مفهومة بدل تجاهله.
- يفصل «فتح الموقع» عن «نسخ الرابط»؛ النسخ لا يفتح تبويبًا جديدًا.
- يصل نوع الإشعار إلى إجراء الخادم باسم `notificationType`، وهو الاسم الذي ينتظره الإجراء الحالي.
- تُعطّل الأزرار حسب حالات مكوناتها الحالية ولا تتغير عقود إجراءات الخادم.

## الاستجابة وإتاحة الوصول

- يبدأ الهاتف من 390px دون تمرير أفقي إجباري.
- أهداف اللمس الأساسية لا تقل عن 44px في المنتج الفعلي، مع تقليل حجم الحاويات لا الأزرار.
- لكل مساحة `aria-label` واضح، ولكل نموذج تسميات مرئية أو متاحة لقارئ الشاشة.
- الحالة لا تعتمد على اللون فقط.
- تحترم الواجهة إعداد `prefers-reduced-motion` الموجود في غلاف الأدمن.

## الاختبارات

- عقد الأربع مساحات والروابط الأساسية.
- تحويل كل قيمة قديمة إلى المساحة الصحيحة.
- ظهور كل عائلات الإجراءات دون فقد الأرشفة أو الحذف أو كلمة المرور.
- فتح بيانات الدخول بدل تنفيذ إعادة ضبط عشوائية من الشريط السريع.
- إرسال `notificationType` الصحيح إلى إجراء الإشعارات.
- عدم فتح رابط الموقع عند الضغط على «نسخ الرابط».
- استمرار اختبارات جدول العملاء والتنقل والصلاحيات الحالية.
- فحص 390×844 و1440×900، ثم typecheck وlint والاختبارات والبناء.

## معايير القبول

- أربع مساحات رئيسية فقط داخل ملف العميل.
- لا تختفي أي وظيفة أو صلاحية أو رابط عميق قائم.
- تعمل الروابط القديمة وتحدد المساحة الجديدة المناسبة.
- لا توجد كلمة مرور أو جلسات خارج مساحة الدعم والحماية.
- كل الإجراءات المتشابهة متجاورة، والإجراءات الحساسة منفصلة بصريًا.
- لا توجد بطاقات داخلية ضخمة أو تمرير أفقي غير مقصود.
- يمر الاختبار المستهدف والاختبارات الكاملة وtypecheck وlint وbuild دون فشل جديد.

## خارج النطاق

- إعادة تصميم قائمة العملاء الخارجية.
- تعديل مخطط Prisma أو دورة الاشتراك التجارية.
- إنشاء نظام صلاحيات جديد أو مشاركة بيانات دخول الأدمن.
- إضافة قنوات تواصل جديدة غير موجودة حاليًا.
