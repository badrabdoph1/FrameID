# Admin Console Deduplication Design

## الهدف

تحويل الأدمن إلى مساحات عمل هادئة وسريعة، بحيث توجد لكل كيان قائمة واحدة ومكان تعديل واحد وإجراء أساسي واضح، من دون فقد أي صلاحية أو مسار أو وظيفة عاملة.

## قاعدة البنية

كل قسم إداري يتكون من:

1. رأس مختصر باسم القسم وإجراء أساسي واحد.
2. مؤشرات قليلة مرتبطة بقرار فعلي.
3. قائمة أو جدول واحد للعناصر.
4. زر تعديل واحد لكل عنصر.
5. محرر واحد يظهر عند اختيار عنصر أو إنشاء عنصر جديد.
6. قائمة إجراءات ثانوية للحالة والأرشفة والنشر مع تأكيد العمليات الخطرة.

صفحات الملخص مثل المال والمحتوى والنظام تعرض حالة وروابط عميقة فقط، ولا تعيد نشر نماذج أو قوائم إدارة كاملة موجودة في الصفحات المتخصصة.

## اللغة والمظهر

تبقى الهوية الداكنة والذهبية الحالية لأنها جزء من نظام الأدمن الموجود. يقل عدد البطاقات والظلال، وتستخدم حدود ومسافات واضحة بدل الزخرفة. توحّد جميع المسميات المرئية بالعربية، مع إبقاء الأكواد والقيم التقنية بالإنجليزية عند الضرورة فقط.

## المسارات والصلاحيات

لا يحذف أي مسار عامل. المسارات القديمة التي تنافس مسارًا أساسيًا تصبح تحويلات متوافقة أو واجهات رقيقة. تبقى حواجز `requireAdminPermission` و`requireSuperAdminSession` وإجراءات الخادم الحالية كما هي ما لم يثبت الاختبار وجود تعارض.

## ترتيب التنفيذ

1. الباقات: النموذج المرجعي للقائمة والمحرر الواحد.
2. صفحة المال: ملخص وقرارات عاجلة فقط.
3. القوالب والمحتوى والوسائط: ملكية واضحة لكل نوع بيانات.
4. العملاء والمواقع: وجهة أساسية واحدة لتفاصيل العميل.
5. الاشتراكات والمدفوعات وإعدادات الدفع.
6. الرسائل والإشعارات والبريد.
7. النظام والعمليات والنسخ الاحتياطي والسجلات.
8. التنقل الموحد واللغة والحالات المتجاوبة.

## الباقات كنموذج

تعرض `/admin/plans` كل باقة مرة واحدة داخل قائمة قابلة للمسح السريع. تحتوي كل باقة على الاسم والسعر والحالة وعدد الاشتراكات وطلبات الدفع، وزر تعديل واحد وقائمة إجراءات. يوجد محرر واحد فقط في DOM، يفتح لإنشاء باقة أو تعديل الباقة المحددة. تغيير الحالة يتم من مسار واحد واضح، وتبقى الأرشفة منفصلة مع تأكيد.

## حالات الواجهة

- التحميل: هيكل ثابت لا يغيّر حجم الصفحة.
- الفراغ: شرح قصير وزر إنشاء العنصر الأول.
- النجاح والخطأ: رسالة أعلى مساحة العمل مرتبطة بالإجراء.
- الهاتف: القائمة أولًا والمحرر في طبقة/قسم كامل العرض، مع أزرار ثابتة الحجم ونصوص لا تتداخل.
- سطح المكتب: قائمة ومحرر في عمودين عند وجود عنصر محدد، وقائمة بعرض كامل عند عدم وجوده.

## معايير القبول

- لا يوجد أكثر من نموذج تعديل واحد للكيان نفسه داخل الصفحة.
- كل عنصر يظهر مرة واحدة وله زر تعديل واحد.
- صفحات الملخص لا تكرر الإدارة التفصيلية.
- لا تضيع وظائف الحفظ والتفعيل والأرشفة والنشر والصلاحيات.
- تعمل المسارات القديمة أو تتحول بوضوح إلى المسار الأساسي.
- جميع المسميات اليومية عربية ومتسقة.
- تمر اختبارات المكونات والإجراءات والتنقل، ثم فحص الهاتف وسطح المكتب.
